## 🧼 1. 비주얼 폴리싱 (Visual Polish)
*2030이 보기에 1픽셀 단위로 미세 조정이 필요하거나, 공장형 앱 느낌을 주는 투박한 스타일링 요소 진단*

### 1-1. 라이트 모드 배경 순백색 — 웜 크림 톤으로 전환 필요

- **대상 파일**: `src/styles/theme.ts:58`
- **현재 코드**:
  ```ts
  BG_LIGHT_SNOW: '#F8F9FA',
  CARD_LIGHT_WHITE: '#FFFFFF',
  ```
- **감성 평론**: "라이트 모드 배경이 병원 검사지 같아요. 인스타 크리에이터들이 피드 배경으로 쓰는 '밀크 화이트' 혹은 '웜 아이보리' 톤이 아닌, RGB 순수 흰색 카드가 눈에 튑니다. 특히 카드 배경 `#FFFFFF`은 다크 모드와의 명도 갭이 너무 크고 밝아서 채팅 피드를 오래 보면 피로해요."
- **디테일 개선안**:
  - `BG_LIGHT_SNOW` → `#FAF8F5` (따뜻한 오프화이트, 크림 언더톤)
  - `CARD_LIGHT_WHITE` → `#FFFDF9` (밀크 아이보리, 순수 흰색 대비 -3% 명도)
  - `LightBlends.BG_SURFACE`(현 `#F5F1F8`) 유지 — 이미 올바른 보라빛 틴트
  - 카드 `borderWidth` 전체를 `StyleSheet.hairlineWidth`(`0.5~1px`) 로 통일해 얇고 세련된 구분선 유지

### 1-2. 채팅 말풍선 모서리 반경 — iOS 네이티브 대비 각진 느낌

- **대상 파일**: `app/(tabs)/chat.tsx` (스타일 섹션 내 `chatBubble` 관련)
- **현재 코드** (추정값, 소스 내 `Radius.lg` = `12px` 적용):
  ```ts
  borderRadius: Radius.lg,  // 12px
  ```
- **감성 평론**: "애플 메시지 앱 말풍선은 꼬리 포함 20px 급의 물방울형 반경을 씁니다. 트윈이 채팅 버블은 `Radius.lg(12px)`라 직사각형 쪽에 가깝고, 인스타 DM 감성과 비교하면 2세대 앞 느낌이에요."
- **디테일 개선안**:
  - 말풍선 `borderRadius` → `18px`
  - 발신 버블 우측 하단, 수신 버블 좌측 하단 모서리만 `4px`로 '말풍선 꼬리'처럼 처리
    ```ts
    // 발신 버블
    borderTopLeftRadius: 18, borderTopRightRadius: 18,
    borderBottomLeftRadius: 18, borderBottomRightRadius: 4,
    // 수신 버블
    borderTopLeftRadius: 18, borderTopRightRadius: 18,
    borderBottomLeftRadius: 4, borderBottomRightRadius: 18,
    ```

### 1-3. 채팅 헤더 글로우 효과 — 오버드라이브 상태

- **대상 파일**: `app/(tabs)/chat.tsx`
- **현재 상태**: 채팅 룸 헤더의 `shadowColor: '#7C3AED'`, `shadowOpacity: 0.6`, `shadowRadius: 24`, `elevation: 20`
- **감성 평론**: "헤더에 네온 보라 그림자가 `opacity 0.6`으로 박혀 있으면, 실제 기기에서 보면 헤더가 떠 있는 것처럼 어색합니다. 인스타그램이나 스레즈 헤더처럼 '거의 안 보이는' 블러 처리 혹은 보더라인 0.5px만으로도 공간 분리가 충분해요. 글로우가 과하면 오히려 저가 앱처럼 보입니다."
- **디테일 개선안**:
  - `shadowOpacity` → `0.18` (기존 대비 ⅓ 수준)
  - `shadowRadius` → `10`
  - `elevation` → `6`
  - 대신 하단 구분선 `borderBottomWidth: StyleSheet.hairlineWidth`, `borderBottomColor: 'rgba(124,58,237,0.18)'` 추가로 공간 분리 보완

### 1-5. 설정 섹션 헤더 레이블 — 스타일 밀도 과다

- **대상 파일**: `app/(tabs)/settings/index.tsx` (각 섹션 헤더 카드 영역)
- **현재 상태**: 카드마다 `borderWidth: 1`, `shadowOpacity: 0.22`, `shadowRadius: 14`, 내부 그라데이션 레이어가 중첩 3~4회 적용
- **감성 평론**: "설정 화면에서 카드 하나하나가 '저를 봐주세요' 하고 튀어나오는 구조입니다. 애플 설정(Settings.app)은 경계선 없이 시스템 배경색 차이만으로 영역을 구분하죠. 트윈이 설정은 시각적 과포화 상태입니다."
- **디테일 개선안**:
  - 보조 설정 카드들(테마 토글, 카카오 동기화, 언어/알림 등)의 `shadowOpacity` → `0.10` 이하로 하향
  - 테마/개인정보 핵심 카드만 `shadowOpacity: 0.18` 유지, 나머지 비핵심 카드는 그림자 제거
  - 카드 간 구분을 `borderWidth` 대신 `marginBottom: 8`과 배경색 차이로 처리하는 섹션 묶음 패턴 도입

## ✂️ 2. UI 다이어트 & 단순화 (UI Simplification)
*유저의 인지적 피로를 유발하거나 터치 흐름을 방해하는 불필요한 스텝 및 인터랙션 진단*

---

### 2-1. 계정 삭제 2단계 모달 플로우 — 경고 텍스트 난해

- **대상 파일**: `app/(tabs)/settings/index.tsx:2398~2451`
- **현재 플로우**:
  1. "계정 삭제" 버튼 탭
  2. 1단계 모달: `"그동안의 기록이 전부 지워집니다."` + `"이어하기 (삭제 절차 진행)"` 버튼
  3. 2단계 모달: 장문의 데이터 나열 + `"⚠️ 확인하였으며, 계정 삭제에 동의합니다"` CTA
- **감성 평론**: "'이어하기 (삭제 절차 진행)'은 약관 동의 버튼 같아서 인스타 세대는 아무 생각 없이 연타합니다. 오히려 중단할 기회를 명확히 줘야 하고, 2단계 확인 문구가 법률 고지처럼 길어서 오히려 '무시하고 싶어지는' 심리 효과가 납니다."
- **디테일 개선안**:
  - 1단계 → 2단계 인터벌 제거, 단일 바텀시트로 통합
  - 경고 핵심 문구 3줄 이내로 요약: `"지도 핀·리포트·DNA 기록이 전부 지워져요. 복구 방법이 없어요."`
  - CTA 텍스트 → `"그래도 삭제할게요"` (법률 문체 → 대화체)
  - 취소 버튼 → `"아니요, 계속 쓸게요"` (이탈 억제 감성 강화)

### 2-2. 채팅 첨부 버튼 → 액션 시트 노출 단계 — 탭 2회 필요

- **대상 파일**: `app/(tabs)/chat.tsx` (AttachmentBar 컴포넌트, `showAttachBar` 상태 토글)
- **현재 플로우**:
  1. 입력창 좌측 `+` 버튼 탭 → `setShowAttachBar(true)`
  2. AttachmentBar 슬라이드업 확인 후 원하는 아이템(사진/영상/위치/선물/카카오) 탭
- **감성 평론**: "기본 iMessage, 인스타 DM 모두 첨부 아이콘을 롱프레스하면 바로 선택지가 뜨거나, 탭 한 번에 팝오버가 뜹니다. 트윈이는 탭 → 확인 → 다시 탭의 2단계인데, 짧은 '선물하기' 같은 충동적 액션에는 1단계가 더 맞아요."
- **디테일 개선안**:
  - `+` 버튼을 `onLongPress` 감지 추가 → 롱프레스 시 AttachmentBar 즉시 팝업
  - 일반 탭은 카메라 앱 바로 실행 (단순 사진 전송이 가장 빈번한 케이스)
  - 카카오톡 업로드처럼 덜 자주 쓰는 항목은 AttachmentBar 내 하단 "더보기(···)" 뒤로 접기

### 2-3. 기억 삭제 체크박스 → "전체 선택" 흐름 — 삭제 버튼 발견이 어려움

- **대상 파일**: `app/(tabs)/settings/index.tsx:876~994`
- **현재 플로우**:

  2. "전체 선택" Pressable 탭
  3. 스크롤 후 하단 "🔥 선택한 기억 N개 영구 파기" 버튼 찾아서 탭

  - 파기 확인 모달 → 바텀시트 형태로 변경 (전체 화면 오버레이 → 하프 시트)

### 2-4. 2FA 설정 3단계(QR → OTP → 백업) — 중도 이탈 시 재시작 구조

- **대상 파일**: `app/(tabs)/settings/security.tsx:516~640`
- **현재 플로우**:
  - `setupPhase: 'qr' → 'otp' → 'backup'` 순차 진행
  - 뒤로가기(`← QR 코드로 돌아가기`) 탭 → `setSetupPhase('qr')`
  - 중도 닫기 → `setShowSetup(false)` → 처음부터 재시작
- **감성 평론**: "QR 스캔 단계에서 인증 앱으로 갔다 와야 하는데, 그 사이에 앱이 백그라운드 가면 모달이 초기화됩니다. 인증 앱을 열고 다시 돌아오면 QR을 처음부터 다시 스캔해야 해요. 이 UX는 2030 특히 스마트폰 전환이 많은 유저에게 진입장벽입니다."
- **디테일 개선안**:
  - `AppState` 리스너로 `foreground` 복귀 시 setupPhase 유지 (state 보존)
  - QR 코드 아래 "인증 앱에서 스캔 후 돌아오세요 ↩" 안내 상주 노출
  - 모달 닫기 시 → `"인증 설정을 중단할까요? 다시 시작해도 돼요."` Alert 중간 확인 추가

### 2-6. 주간 리포트 모달 — 스크롤 길이 과다, 섹션 접힘 없음

- **대상 파일**: `src/components/chat/WeeklyReportModal.tsx`

## 📐 4. 터치 & 인터랙션 피드백 품질 (Touch UX)

### 4-1. 메모리 항목 체크박스 — 터치 타겟 너무 작음

- **감성 평론**: "2030 유저는 엄지 탭을 쓰는데, 체크박스가 20px 정사각형이면 연속 탭 시 오선택이 발생합니다. 애플 HIG 권장 최소 탭 영역은 44×44pt입니다."

  - `Pressable`에 `hitSlop={{ top: 12, bottom: 12, left: 16, right: 16 }}` 추가
  - 체크 표시(✓) 폰트 크기 `14px` 유지, 내부 패딩으로 조정

- **현재 상태**: 전송 버튼이 `Pressable` 또는 `TouchableOpacity`이나 `activeOpacity` 명시 없음 (기본값 0.2 적용)
- **감성 평론**: "전송 버튼의 투명도 딥이 0.2(80% 투명)까지 내려가면 '버튼이 사라지는' 것처럼 보여서 초심자 유저가 탭이 잘 됐는지 불안해합니다. 인스타 DM 전송 버튼의 스케일 피드백(0.92배 축소)이 훨씬 자연스러워요."
- **디테일 개선안**:
  - `TouchableOpacity activeOpacity={0.72}` 또는 `Pressable`에 `withTiming(0.92)` scale 피드백 추가
  - 전송 성공 시 0.3초 초록빛 플래시 또는 체크 아이콘 전환 모션 추가 (전송 확인 피드백)

## 🚨 7. 접근성 & 인지 부하 (Accessibility & Cognitive Load)
*시각·인지 부하 관련 잠재 이슈 점검*

---

### 7-1. 네온 글로우 버튼 라이트 모드 대비 — 가독성 이슈

- **대상 파일**: `app/(tabs)/settings/index.tsx` (구독 플랜 카드), `src/components/chat/WeeklyReportModal.tsx`
- **현재 상태**: 라이트 모드에서도 `#7C3AED`(네온 바이올렛) 위에 흰색 텍스트 렌더
- **진단**: WCAG AA 기준 최소 대비율 4.5:1 충족 여부 의심
  - `#7C3AED` 위 `#FFFFFF` 대비율: 약 5.5:1 (간신히 통과)
  - `#D946EF` 위 `#FFFFFF` 대비율: 약 3.3:1 (AA 미달 ⚠️)
- **디테일 개선안**:
  - 그라데이션 중간색 `#D946EF`(오키드 마젠타) 위의 버튼 텍스트는 `#0F172A`(짙은 네이비) 또는 `#FFFFFF`에 `textShadow` 강화
  - 라이트 모드 전용 CTA 버튼: 배경을 파스텔 그라데이션(`#FFB7CE → #B39DDB`)으로, 텍스트는 `#2D1B5A` 다크 바이올렛으로 변경

---

### 7-2. 작은 폰트 10px 남용 — 모바일 최소 권장 크기 미달

- **대상 파일**: `app/(tabs)/chat.tsx:1258`, `app/(tabs)/settings/index.tsx:163`
- **현재 상태**:
  ```ts
  fontSize: 10  // 히스토리 탭 카테고리 칩, 채팅 탭 시간 표시
  ```
- **진단**: 모바일 접근성 가이드 권장 최소 폰트 크기는 12px (또는 11sp Android), iOS HIG는 11pt 권장
- **디테일 개선안**:
  - 10px 폰트 전체 `11px` 이상으로 상향 (단 고밀도 상단 바 등 구조상 불가피한 경우 제외)
  - 보조 텍스트의 경우 `opacity: 0.65`보다 명도 기반 컬러 토큰(`t.textMuted`)을 사용해 투명도 없이 가독성 확보

---

### 7-3. 연속 진동(햅틱) 빈도 — 짧은 시간에 과다 발동

- **대상 파일**: 복수 파일 (`chat.tsx`, `settings/index.tsx`, `CoursePlanner.tsx` 등)
- **현재 상태**: 버튼 탭마다 `Haptics.impactAsync(Light)` + 성공 시 `notificationAsync(Success)` 연속 발동
- **감성 평론**: "햅틱 피드백이 모든 탭에서 터지면, 진동이 너무 잦아서 '묵직한 확인감' 대신 '뭔가 계속 울리네'로 느껴집니다. 중요 액션(전송, 삭제 확인, 잠금 해제)에만 Medium/Heavy를 쓰고, 일반 탭은 Light를 쓰되 연속 탭에는 Debounce를 걸어야 해요."
- **디테일 개선안**:
  - 일반 탭: `Haptics.selectionAsync()` (가장 가벼운 레벨)
  - 토글/슬라이더: `Haptics.impactAsync(Light)`
  - 전송/저장 완료: `Haptics.impactAsync(Medium)`
  - 삭제/계정 파기 최종 확인: `Haptics.impactAsync(Heavy)` + `notificationAsync(Warning)`
  - 100ms 이내 연속 햅틱 Debounce 처리
