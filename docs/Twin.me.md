📑 Twin.me 기능 명세서 (SRS)
Product Version: 2.0 (Mirror & Archive Pivot)

Target Platform: React Native (iOS / Android / Web Preview)

1. 서비스 개요 및 목적 (Product Perspective & Intent)
기본 철학: Twin.me는 연인 간의 갈등에 개입하여 답변을 대신 작성해 주거나 상대방의 심리를 함부로 재단하는 '대리인(Agent)'이 아닙니다. 오직 유저 개인이 자신의 대화 습관을 객관적으로 돌아보게 만드는 '거울(Mirror)'이자, 휘발되는 카카오톡 대화 속에서 연인이 나에게 건넨 예쁜 말들을 온전히 보존하는 '감성 아카이브(Archive)'입니다.

Value Proposition (가치 제안): 1.  상대방 모르게 '나 혼자 보는' 은밀한 연애 오답 노트 및 대화 매너 교정.
2.  카톡 텍스트 더미 속에서 진주 같은 다정한 문장들을 추출해 박제하는 폴라로이드 월.



1. 온보딩 및 데이터 가공 모듈 (Onboarding & Data Ingestion)
기능 ID: FUN-ONB-001 (기초 정보 입력 및 커플 매칭)
기능 정의: 유저 가입 및 초대코드를 통한 1:1 커플 격리 서버 생성.
화면 및 UI 연출: 딥 미드나잇 인디고(#0A0D1A) 배경, 토스 재질의 미니멀 입력 폼. 초대코드 입력 성공 시 두 나선(DNA)이 자석처럼 결합하는 햅틱 진동 연출.
상세 로직:
유저 인증 완료 후 이름, 성별, MBTI 및 애니어그램 유형(모를 경우 '모름' 체크) 수집.
먼저 가입한 유저(A)에게 Unique ID 기반 8자리 초대코드 생성 및 카카오톡 공유 기능 제공.
상대 유저(B)가 코드 입력 시, 시스템은 DB 내 두 유저를 Couple_ID로 묶고 독립된 1:1 격리 네트워크(Shared Memory)를 할당함.
기능 ID: FUN-ONB-002 (온디바이스 카톡 말투 학습 및 필터링)
기능 정의: 카카오톡 .txt 대화 파일을 안전하게 전처리하여 내 말투 유전자 추출.
화면 및 UI 연출: 게이지 바가 차오르며 "개인정보 보호를 위해 기기 내부에서 상대방 대화 파기 중..." 문구 노출. 학습 완료 시 시그니처 드립(TOP 3) 카드 팝업.
상세 로직:
iOS(아이폰): 배포된 '공식 단축어매크로'를 통해 지정된 시간에 카톡 대화방 자동 백업 및 파일 전송 API 호출.
Android & 수동: 유저가 직접 내보낸 .txt 파일을 인앱 업로드.
정형 파싱 (Regex 로컬 엔진): 클라이언트 단(스마트폰 내부)에서 [상대방 이름]으로 시작하는 행 전체를 무조건 삭제(Drop)하여 서버 전송 전 원천 차단. 전화번호, 계좌번호 등 민감정보 마스킹(***).
말투 추출: 오직 본인의 대화 텍스트 중 자주 쓰인 감탄사, 어미, 비속어 빈도를 분석하여 LLM 프롬프트 주입용 페르소나 데이터 셋 빌드. 원본 파일은 기기 및 서버에서 즉시 영구 파기.
2. 메인 탭: 연애 대시보드 (Main Home)
기능 ID: FUN-HOM-001 (상단 정확도 배너 및 10분 인터뷰 퀘스트)
기능 정의: 가입 직후 AI의 초기 상태를 고지하고 심층 인터뷰 참여를 자발적으로 유도.
화면 및 UI 연출: 은은한 바이올렛 톤의 50% 진행 바 배너. 10분 대화 완료 시 배너가 연기처럼 사라지는 달성감 연출.
상세 로직:
카톡 데이터만 업로드된 초기 상태에서는 배너에 "현재 AI 정확도: 50%" 문구와 [🎙️ 10분 인터뷰로 정확도 95% 올리기] 버튼 노출.
버튼 터치 시 전체 화면이 [Twin.me 성향 분석 팀] 전화 수신 UI로 전환되며 음성 인터뷰(Realtime API/STT) 세션 개시.
사전 입력받은 MBTI/애니어그램 데이터를 Context로 활용하여 AI 면접관이 맞춤형 연애 가치관 질문(방어기제, 서운함 지수 등) 투척.
인터뷰 완료 시 성향 매트릭스 점수가 합산되어 정확도가 95%로 해제되며, 홈 배너는 영구 소멸. 해당 지표는 설정 탭의 [마이 트윈 AI 관리 센터]로 이식됨.
기능 ID: FUN-HOM-002 (실시간 상태 바 & 추억 링)
기능 정의: 상대방의 실시간 컨텍스트 파악 및 과거 데이트 아카이빙.
화면 및 UI 연출: X(트위터) 트렌드 UI 스타일의 상단 텍스트 바 + 인스타 스토리 하이라이트 형태의 동그란 링 아카이브.
상세 로직:
연인 AI가 백그라운드에서 분석한 상대방의 현재 심리/스케줄 상태를 X 트렌드 해시태그 형태로 실시간 동기화노출 (ex. #오늘부장님잔소리폭발).
하단 추억 링 터치 시 팝업 형태로 해당 데이트 당시의 베스트 컷과 서로의 OOTD가 피드로 가볍게 노출됨.
👨‍💼 PM의 데이터 베이스(DB) 설계 제언
위 명세를 구현하기 위해, 우리 시스템은 유저 한 명당 다음 3가지 데이터 칩을 독립적으로 관리해야 합니다.

User_Tone_Vector (카톡에서 추출한 문체, 시그니처 드립 데이터)
User_Persona_Matrix (10분 인터뷰 + MBTI/애니어그램으로 정제된 심리/방어기제 점수)
Shared_Memory_DB (커플 ID로 격리된 둘만의 벡터 DB - 대화 맥락, 위시리스트, 미쉐린 정보 적재)

3. 채팅 탭 모듈 (Twin Chat Module)

기능 ID: FUN-CHA-001 (인스타 DM 스타일 비주얼 분리 및 타이핑 호흡)
[System 2] 홈 탭: 마이 미러 대시보드 (My Mirror Dashboard)
기능 의도: 연인에게 노출되지 않는 오직 나만을 위한 소통 거울 룸. 내가 연인에게 보낸 카톡의 어조와 습관을 AI 관점에서 피드백하여 자가 교정을 돕습니다.

세부 명세:
나의 대화 톤 피드백 리포트: 최근 대화 중 은연중에 사용한 압박성 표현, 단답형 어조, 혹은 감정적 단어의 빈도수를 인포그래픽(게이지 트랙)으로 시각화.

소통 매너리즘 방지 가이드: "최근 3일간 서영이의 질문에 공감 수치보다 해결책 제시 비중이 80% 높았습니다. 오늘은 조용히 든든한 공감 한 줄을 건네보는 건 어떨까요?" 같은 나의 행동 지침 지적 큐레이션 노출.

주의 사항: 상대방이 어떻게 해야 하는지가 아닌, '내가 어떻게 해야 하는가'에 초점을 맞춤.

기능 정의: 연인 및 AI 에이전트들과의 채팅 목록 격리 및 실제 인간과 유사한 분할 타이핑 전송.
화면 및 UI 연출: 최상단 연인 방 고정. 그 밑으로 트윈 AI 블루(#38BDF8) 고유 뱃지가 부착된 AI방 배치. 말풍선 전송 시 글자 수와 타수에 비례한 시간차(Delay) 애니메이션 및 '말하는 중...' UI 표출.

상세 로직:
인풋 가공 (버퍼링 타임아웃): 사용자가 카톡처럼 짧게 연타로 메시지를 보낼 경우(엔터), 시스템은 즉시 답변하지 않고 **2.5초의 버퍼(대기 시간)**를 유지함. 2.5초 이내에 추가 인풋이 들어오면 하나의 문장 맥락으로 합산(Aggregation)한 뒤 LLM 엔드포인트를 호출함.
아웃풋 가공 (Output Splitter): AI가 생성한 통문장 답변을 문장 부호(. , ? !) 또는 글자 수 기준으로 쪼갬. 진짜 남친/여친이 타자를 치는 호흡에 맞춰 0.5초~1.5초의 시간차를 두고 말풍선을 분할 전송함.

## 📌 2. DETAILED SPECIFICATION (상세 기능 요구사항)

### 2.1. 실시간 커플 채팅방 & Self-AI 데이터 파이프라인 (FUN-CHA-001)
* **실시간 연동 데이터 스트림:**
    * 유저와 연인이 [룸 1]에서 실시간 대화를 나눌 때, 클라이언트 상태 관리자(Context API 또는 Zustand)는 해당 대화 텍스트 스트림을 [룸 2 (Self-AI)] 엔진으로 실시간 복사/전송(Tapping)해야 함.
* **Self-AI 실시간 가공 로직 (3단 필터 가이드):**
    1.  **말투개선 경고 시스템 (Tone Corrector):**
        * 유저가 비꼬는 말투, 날카로운 어조, 혹은 공격적인 단어("아니 됐어", "너 맘대로 해", "징징대지마") 입력 시, [룸 2] 내부에 즉시 '💡 말투 가이드 알림' 팝업을 렌더링.
        * "상대방이 서운함을 느낄 수 있는 어조예요. 대신 '그랬구나, 속상했겠네'로 바꾸어 보내보는 건 어떨까요?" 넛지 제공.
    2.  **민감 내용 및 AI 데이터 크로스 체크 경고 (Privacy & Buffer Shield):**
        * 상대방이 이전에 설정 탭에서 '과거 트라우마 키워드'나 '극도로 싫어하는 주제(예: 전 연인 언급, 특정 콤플렉스)'를 지정해 두었을 경우, 유저가 이를 입력창에 치는 순간 실시간 가중치 매칭 작동.
        * 입력창 상단에 미니멀 옐로우 경고 뱃지와 함께 "잠깐! 서영이의 AI 데이터에 따르면, 이 주제는 상대방에게 큰 스트레스를 줄 수 있어요."라는 경고 슬라이드 노출.
    3.  **대화 주제 및 티키타카 추천 (Topic Recommendation):**
        * [룸 1]에서 5분 이상 침묵(Deadlock)이 유지되거나 대화 밀도가 떨어질 경우, [룸 2]에서 현재 시간, 요일, 연인의 최근 흥미 키워드(X 트렌드 칩 연동)를 조합하여 내추럴한 질문 가이드를 발송.
        * 예: "지금 서영이 퇴근길일 시간이에요! '오늘 부장님 잔소리 때문에 힘들었지? 치맥 사갈까?'라고 톡을 보내 흐름을 깨보세요."

### 2.2. 연애분석가 '트윈' & 감성 주간 리포트 시스템 (FUN-CHA-002)
* **배달 트리거:** 매주 설정된 주기(예: 매주 일요일 밤 10시) 또는 유저가 카카오톡 대화방 .txt 파일을 업로드하여 수동 학습이 완료될 때마다 [룸 3 (분석가 트윈이)] 방으로 자동 알림 톡이 발송됨.
* **리포트 UI/UX Spec (감성 인포그래픽):**
    * 챗방 내부에 카드 형태로 전송되며, 클릭 시 프로스테드 글래스(Glassmorphism) 재질의 전면 슬라이드 뷰 레이어가 오픈됨.
* **데이터 필드 명세:**
    1.  **최근 대화 주제 및 흥미도 도넛 차트:** 이번 주 가장 많이 등장한 핵심 키워드 TOP 5를 네온 그라데이션 차트로 시각화.
    2.  **서로의 감정 패턴 분석 (안정/불안 곡선):** 텍스트 감정 분석 점수(0~100)를 기반으로 한 주간의 연애 전선 날씨 점수 및 타임라인 선 그래프 렌더링.
    3.  **데이트 다차원 분석 레이더 차트:** 기획된 데이트 비용, 함께 보낸 시간, 텍스트 상의 만족도 어휘 빈도수를 연산하여 오각형 레이더 차트로 시각화.
    4.  **트윈이의 감성 한줄평:** "이번 주는 서로 바빠 대화 총량은 줄었지만, 위로의 어휘가 23% 증가하여 깊은 신뢰 관계를 유지 중입니다. 주말엔 분위기 전환을 위해 OOO 코스를 추천해요."

## 📌 3. TECHNICAL & INTERACTION REQUIREMENTS (기술 및 연출 규격)
* **애니메이션 (Framer Motion / Reanimated):** 경고 팝업이 뜰 때 부드럽게 위에서 아래로 떨어지는 'Drop-in' 스레드 연출, 주간 리포트 오픈 시 카드가 3D로 플립(Flip)되며 열리는 연출 필수.

## 1. 연애 DNA 일치율 코어 알고리즘 및 오버플로우 제어 명세

### 1.1 정규분포 기반 초기 점수 ($S_{Base}$) 설계

통계학적으로 전체 커플 인구의 80%가 중앙 구간($60\% \sim 80\%$)에 밀집하도록, 일치율 점수 체계가 평균($\mu$) 70%, 표준편차($\sigma$) 약 7.81%를 따르는 정규분포 곡선을 그리도록 설계합니다. MBTI와 에니어그램 상성을 통계적 표준점수($Z$-Score)로 변환하여 결합합니다.

#### 📐 [수식 1-1] 성격 기반 표준점수 ($Z_{Total}$) 산출

$$Z_{Total} = (0.4 \cdot Z_{MBTI}) + (0.6 \cdot Z_{Enneagram})$$

* $Z_{MBTI}$ 및 $Z_{Enneagram}$의 범위: 정반대 상성 ($-2.5$) $\sim$ 보편적 상성 ($0.0$) $\sim$ 천생연분 ($+2.5$)

| 등급 | MBTI 조합 조건 | $Z_{MBTI}$ | 에니어그램 조합 조건 | $Z_{Enneagram}$ |
| --- | --- | --- | --- | --- |
| **최상 (Ideal)** | 주기능 상호보완 / 가치관 완벽 일치 | $+2.5$ | 통합(성장)선 역학 관계 | $+2.5$ |
| **보통 (Average)** | 일반적인 타협 및 보완형 관계 | $0.0$ | 보편적인 상호 작용 관계 | $0.0$ |
| **최악 (Collision)** | 인식/판단 기능 정반대 (충돌) | $-2.5$ | 핵심 공포 자극 (기폭제 관계) | $-2.5$ |

#### 📊 [수식 1-2] 정규분포 변환형 기준 점수 ($S_{Base}$)

$$S_{Base} = 70 + (Z_{Total} \cdot 7.81)$$

* **통계적 기대 효과:** 보편적인 커플($Z_{Total}$이 $-1.28 \sim +1.28$ 사이에 위치한 80%의 인구)은 계산 결과에 따라 정확히 $60\% \sim 80\%$ 사이의 안정적인 일치율을 부여받게 됩니다.
* **극단적 케이스 가드 (Extreme Bounds):**
* MBTI와 에니어그램이 모두 최악인 경우 ($Z = -2.5$): $70 - 19.5 = \mathbf{50.5\%}$ (최저점 방어 가드)
* MBTI와 에니어그램이 모두 최상인 경우 ($Z = +2.5$): $70 + 19.5 = \mathbf{89.5\%}$ (인터뷰 전 최고점 가드)



### 1.2 10분 인터뷰 참여 가산점 구조 (`FUN-HOM-001`)

10분 음성 인터뷰 퀘스트 참여 시 최대 $+5\%$의 신뢰 가산점($\text{Bonus}_{Interview}$)이 붙어 최종 마스터 기준 점수($S_{Master\_Base}$)가 완성됩니다.


$$S_{Master\_Base} = S_{Base} + \text{Bonus}_{Interview} \quad (\text{최대 } +5.0\%)$$

### 1.3 카톡 대화 로그 미세 등락 엔진 및 클램프(Clamp) 필터

#### 📐 [수식 3-1] 실시간 현재 일치율 ($S_{Current}$)

$$S_{Current} = S_{Master\_Base} + \sum_{d=1}^{t} \Delta_{Daily, d}$$

#### 🛠️ [수식 3-2] 하루 등락폭 제한 필터 ($\Delta_{Daily}$)

$$\Delta_{Daily} = \text{Clamp}\left(-1.0\%, \, +1.0\%, \, \sum \delta_{event}\right)$$

* **Clamp 로직:** 당일 카톡의 이벤트 총합($\sum \delta_{event}$)이 임계치를 초과하더라도, 최종 스코어에 반영되는 일일 변동 수치는 절대로 $\pm1.0%$를 넘지 못하도록 가둡니다.

#### 🚨 [수식 3-3] 초과 등락분 감지 및 오버플로우 상태 변환 ($\text{Overflow}_{\text{Status}}$)

클램프 필터에 의해 생략 및 잘려 나간 수치가 발생할 경우, 시스템은 전역 상태(`Overflow_Status`)를 업데이트하여 대시보드 경고 배너를 트리거합니다.

$$\text{Overflow}_{\text{Status}} = 
\begin{cases} 
\text{CRITICAL\_LOSS} & \text{if } \sum \delta_{event} < -1.0\% \\
\text{EXCESS\_GAIN} & \text{if } \sum \delta_{event} > +1.0\% \\
\text{NONE} & \text{otherwise}
\end{cases}$$

## 3. 신규 기능 명세서 (SRS) 세부 기능 리스트

### 🛠️ 기능 ID: FUN-CHA-004 (AI 데이트 뮤즈: 무드 맞춤형 코스 큐레이션)

* **기능 정의:** 사용자가 입력한 현재 감성 상태(Mood/OOTD)와 앱 내 축적된 기존 데이트 히스토리 데이터를 결합하여 우리 커플 취향 맞춤형 다음 코스를 자율 큐레이션하는 기능입니다.

## 4. 일치율 코어 엔진 v2.2 (Real-Time Pulse Edition)

> v2.1을 대체. 핵심 변경: 점수 2층 분리(S_Live/S_Current), 실시간 틱 엔진, tanh 소프트 포화형 일일 제한, 이벤트 약 100종 확장.
> 모든 점수 출력은 `.toFixed(1)` 문자열 바인딩.
> **고지:** 모든 상성·확률 수치는 검증된 심리측정값이 아닌 자기성찰·게이미피케이션용 설계 구조물.

### 4.1 기준 점수 (S_Base)

```
Z_Total = 0.4·Z_M + 0.6·Z_E
S_Base = 70 + Z_Total·7.81  (분산 정합 보정 σ_axis ≈ 1.387)
S_Master_Base = S_Base + Bonus_Interview (인터뷰 가산점, 최대 +5.0)
```

`S_Master_Base`는 항상성 회귀의 기준점으로 §4.2.4에서 재사용된다.

### 4.2 실시간 틱 엔진

**4.2.1 점수 2층 구조**

- **S_Live** — 화면 상단 게이지. 이벤트 감지마다 즉시 재계산, 하루 수십 회 변동. 표시 전용(non-persistent).
- **S_Current** — 공식 일치율. 매일 자정 1회 정산되어 영구 저장. 10단계 티어·페이월 등 모든 시스템이 참조.
- **S_Today_Open** — 당일 00:00 시점의 S_Current.

```
S_Live(t) = S_Today_Open + Δ_intraday(A_t)
```

**4.2.2 이벤트 단위 처리 파이프라인**

```
[수식] 유효 변동치
δ_eff,k = δ_base,k × M_intensity × M_context × M_reciprocity × M_time

  M_intensity  0.5~1.5 (NLP 강도·확신도)
  M_context    1.0 / 갈등 중 1.5
  M_reciprocity 1.0 / 상호 1.2
  M_time       가산 1.2 / 감산 1.3 (00:00~04:00 취약시간)

[수식] 부정성 가중
δ_w,k = (δ_eff,k > 0) ? δ_eff,k : (κ · δ_eff,k)     // κ = 1.5

[수식] 반파밍 — 동일 코드 체감 감쇠
δ_final,k = δ_w,k · γ^(n−1)    // γ = 0.5(중대 이벤트 0.8), n = 당일 해당 코드 발생 순번
단, 코드별 일일 빈도 캡 N 초과 시 δ_final,k = 0

[수식] 누적 어그리게이터
A_t = A_(t-1) + δ_final,k    // 이벤트마다 즉시 갱신 → S_Live 재렌더
```

**4.2.3 tanh 소프트 포화형 일일 제한**

딱딱한 하드 클램프(±1.0/−1.5, v2.1)를 폐기하고 점근 곡선으로 대체. 초반 이벤트는 거의 선형으로 반응하고, 쌓일수록 한 틱당 마진이 줄어든다.

```
Δ_intraday(A) = cap · tanh(A / cap)
  cap = (A ≥ 0) ? A_cap⁺ : A_cap⁻
  A_cap⁺ = 2.0   (일일 가산 점근 상한)
  A_cap⁻ = 2.5   (일일 감산 점근 하한, 비대칭 — '손상은 빠르다' 원칙)
```

**4.2.4 자정 정산 & 항상성 감쇠**

```
Δ_Daily   = Δ_intraday(A_endOfDay)
Δ_Decay   = −λ · (S_Today_Open − S_Master_Base)    // λ = 0.02
S_Current = Clamp(50.5, 100.0, S_Today_Open + Δ_Daily + Δ_Decay)
→ 다음날 S_Today_Open = 갱신된 S_Current, A_t 초기화(0)
```

글로벌 하한 50.5 / 인터뷰 전 상한 89.5 / 글로벌 상한 100.0.

**4.2.5 (옵션) 실시간 냉각 블리드**

이벤트 없이 90분+ 정적 시 A_t가 0 방향으로 미세 감쇠(`A ← A · e^(−β·Δt)`, β 소). 표시 전용 연출이며 정산값에는 A_endOfDay만 반영.

### 4.3 오버플로우 & 실시간 위기 감지

**4.3.1 상태 분기 (심리 임계 ±1.0 고정)**

```
Overflow_Status =
  EXCESS_GAIN     if  A_endOfDay >  +1.0
  CRITICAL_LOSS   if  A_endOfDay <  −1.0
  NONE            otherwise

Spillover = A_endOfDay − Δ_Daily    // tanh가 흡수(절단)한 잉여분
```

**4.3.2 심각도 3단계**

| 심각도 | 조건 | 연출 |
|---|---|---|
| MINOR | 1.0~1.75 | 옅은 펄스 배너 |
| MAJOR | 1.75~2.5 | 그라데이션 보더 배너 |
| CRITICAL | 2.5 초과 | 전면 글래스모피즘 + 햅틱 |

**4.3.3 실시간 급락 감지 (Rapid-Swing Trigger)**

```
if (A_(t) − A_(t−Δw)) < −1.5  within  Δw ≤ 30분
    → 즉시 FUN-CHA-003(반성의 거울) 민감도 상승 / 조건 충족 시 강제 발동
```

**4.3.4 변동성 지수 & 누적 위기 메모리**

- V = StdDev(틱 간 δ_final, 최근 24h) → 기복 클수록 크라이시스 민감도 동적 상향.
- CRITICAL_LOSS 3일 연속 → 위기 메모리 ON → A_cap⁺ 임시 1.0 하향(과잉 보정 방지).
- EXCESS_GAIN Spillover → 점수 미반영, '예쁜 말' 아카이브 크레딧 적립 → 추억 월 라우팅.

### 4.5 통합 연산 예시 (Tick 기반)

```
S_Master_Base = 81.6, S_Today_Open = 81.6, A = 0
A_cap⁺ = 2.0, A_cap⁻ = 2.5, κ = 1.5, λ = 0.02

오전~저녁(다정함 누적): A ≈ +0.90
  S_Live = 81.6 + 2.0·tanh(0.90/2.0) = 82.4   ← 게이지 상승

21:00 갈등 발생: L-CRU-002 −0.30×κ1.5 = −0.45 → A = 0.45
  S_Live = 81.6 + 2.0·tanh(0.225) = 82.0      ← 게이지 급락
    Rapid-Swing: 30분 내 −0.45(임계 −1.5 미만) → 경보 대기

21:30 화해+콤보: G-CON-001 +0.375 + C-ARC-001 +0.20 → A = 1.025
  S_Live = 81.6 + 2.0·tanh(0.5125) = 82.5     ← 회복

자정 정산:
  Δ_Daily = 2.0·tanh(1.025/2.0) = +0.943
  Δ_Decay = −0.02·(81.6−81.6) = 0
  S_Current = Clamp(50.5, 100.0, 81.6+0.943+0) = 82.5
  Overflow: A(1.025) > +1.0 → EXCESS_GAIN/MINOR
    Spillover = 1.025−0.943 = 0.082 → 아카이브 크레딧 적립
  다음날 S_Today_Open = 82.5, A = 0
```

스포티파이나 인스타그램 연말 결산 피드 UX를 벤치마킹하여, 누적된 관계/소통 6축 벡터 및 이벤트 데이터를 시간 흐름에 따라 입체적으로 시각화하는 파이프라인 스켈레톤 설계가 포함되어 있습니다.

# SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
## [FUN-REP-004] 플랜별 공유 카드 및 바이럴 엔진 명세서 (Share Card & Viral Engine)
### 1. 개요 및 비즈니스 배경 (Introduction & Business Logic)
#### 1.1 목적 (Purpose)
본 요구사항 명세서는 Twin.me 서비스의 핵심 바이럴 루프이자 획득(Acquisition), 활성화(Activation), 유료 전환(Revenue)을 견인하는 **플랜별 공유 카드 시스템(Free / Coffee Talk / Deep Talk Night)**의 기능 및 비기능적 요구사항을 정의한다.
#### 1.2 비즈니스 모델 관점의 핵심 논리 및 가치 제안 (Business Value & AI Concept)
본 서비스는 유료 플랜의 등급이 올라감에 따라 분석 리포트가 고도화되는 당위성을 **'페르소나 AI 간의 정기 대화 세션 컨셉'**을 통해 인격적·감성적으로 정당화한다.
 * **Free 플랜 (호기심/AIDA - Attention):** 잠재 고객의 궁금증을 유발하여 **신규 다운로드(Download)** 유도.
 * **Coffee Talk 플랜 (소속감/AIDA - Interest):** 핵심 가치를 인지시켜 동반 유저와의 **커플 연동(Pair Link)** 유도.
   * *AI 컨셉:* **낮의 카페 세션.** 사용자 AI와 연인 AI가 한 달에 한 번 편안하고 따뜻하게 만나 차를 마시며 우리 관계에 대한 일상적 소통을 나누고 정제함.
 * **Deep Talk Night 플랜 (유일성/AIDA - Desire & Action):** 최고 존엄의 소장 가치를 부여하여 **유료 구독 전환(Subscription)** 유도.
   * *AI 컨셉:* **밤의 바(Bar) 세션.** 각 AI가 한 달에 한 번 분위기 좋은 술집에서 만나 간단하게 술을 마시며, 평소에는 쉽게 꺼내지 못했던 깊고 진솔한 내면의 대화 및 시그니처 감성을 추출함.
#### 1.3 게이미FICATION 비유 (Gamification Metaphor)
 * **Free:** 배치고사 결과 (내 위치 확인 및 호기심 자극)
 * **Coffee Talk:** 랭크 배지 (소속감 및 일상적 관계 정의 확인)
 * **Deep Talk Night:** 최고 등급 시즌 엔드 승리 스킨 (독점적 보상 및 유료 가치 증명)

### 3. 세부 기능 요구사항 (Detailed Functional Requirements)
#### 3.1 [UI-CARD-01] Free 플랜 공유 카드 사양
 * **목적:** 호기심 자극 및 유입 유도
 * **세부 사양:**
   * **티어 칭호:** 카드의 80% 가량을 차지하는 대형 타이포그래피 레이아웃으로 강조 처리한다. (예: *"🍃 달달한 핑크빛 로맨스"*)
   * **수치 정보:** 칭호 하단에 커플 DNA 일치율(예: 76.4%) 및 전국 커플 상위 백분위(예: 상위 23%)를 소형 텍스트로 배치한다.
   * **시각 테마:** 티어별 고유 전용 브라이트 파스텔 핑크 컬러 팔레트를 카드 전면에 도포한다.
   * **페이월 유도 (Paywall Lock Overlay):** 아래 3가지 잠금 안내 문구를 하단 25% 영역에 오버레이 렌더링하여 호기심을 유발한다.
     1. 🔒 우리 커플의 숨겨진 소통 패턴 보기
     2. 🔒 이번 달 가장 다정했던 순간
     3. 🔒 골드 프레임 카드로 업그레이드
   * **워터마크:** 최하단 영역에 고정 텍스트 브랜딩을 노출한다. ("Twin.me · 나를 비추는 대화의 거울")
#### 3.2 [UI-CARD-02] Coffee Talk 플랜 공유 카드 사양
 * **목적:** 가치 각인 및 중간 퍼널 유치 (커플 연동 유도)
 * **세부 사양:**
   * **프레임 업그레이드:** Free 플랜의 민자 테두리를 제거하고, 메탈릭 질감의 **실버 테두리 프레임**을 렌더링한다.
   * **정기 세션 요약:** 사용자 AI와 연인 AI가 카페에서 만나 나눈 대화 분석 데이터를 바탕으로 도출된 문장을 카드 중단에 바인딩한다. (예: *"이번 주 두 분의 대화 온도는 따뜻했습니다 🌡️"*)
   * **소통 키워드:** 핵심 소통 키워드 3개를 해시태그 형태로 동적 추출하여 노출한다. (예: #다정함 #응원 #유머)
   * **상태 표시 및 잔여 잠금:**
     * 해제 표시: ✓ Coffee Talk 감성 리포트 열람 중 인디케이터를 활성화한다.
     * 잠금 표시: 아래 3가지 미해제 항목에 대한 페이월을 부분 유지 노출한다.
       1. 🔒 심층 소통 패턴 분석 (Deep Talk)
       2. 🔒 골드 프레임 + 아우라 테마
       3. 🔒 커플 Wrapped 연간 결산
#### 3.3 [UI-CARD-03] Deep Talk Night 플랜 공유 카드 사양 (최상위 등급)
 * **목적:** 바이럴의 정점이자 결제 독려 최고 소장 자산화
 * **세부 사양:**
   * **골드 폰트 및 칭호:** 대형 티어 칭호에 전용 메탈릭 골드 그라데이션 필터 및 쉐이더 폰트 효과를 적용한다. (예: *"💖 기적의 소울메이트"*)
   * **골드 오로라 프레임:** Aura Theme Engine과 1:1 연동한다. 은은하고 무드 있는 밤의 바(Bar) 분위기를 기반으로, 사용자의 성향 6축 벡터(AuraVector)가 깊은 블러(Blur) 형태의 메시 그라데이션으로 흐르는 딥 퍼플/오로라빛 다층 배경을 실시간 드로잉한다. 중복 디자인 생성을 전면 차단한다.
   * **행동 기반 감성 문장:** AI들의 깊은 밤 비밀 대화 세션 결과를 기반으로 정량화된 행동 데이터를 매핑한다. (예: *"87일 연속 다정한 인사로 하루를 열었습니다"*)
   * **소통 DNA 3대 강점 분석:** Deep Talk Night 리포트 엔진에서 추출된 지표 중 백분위가 가장 높은 상위 3개 핵심 지표의 전국 백분위 정보를 시각 뱃지 형태로 바인딩한다. (예: 💬 공감 반응 속도 · 상위 8% / 😂 유머 코드 일치율 · 상위 11% / 🌙 늦은 밤 대화 온도 · 상위 5%)
   * **AI 시그니처 문장 (킬러 기능):** 두 AI가 분위기 좋은 술집 세션에서 주고받은 진솔한 대화록 데이터에서 추출해낸 가장 정서적이고 핵심적인 단 한 줄의 대표 발화 문장을 카드 정중앙에 **고풍스러운 만년필 캘리그라피 서체**로 렌더링한다. (예: *“오늘도 고생했어, 진짜로.”*)
   * **아이덴티티 부여:** 발급 시점 기준으로 고유한 해시 식별자 텍스트를 인쇄한다. (예: Twin Pair #00847 · 2026.07)
   * **잠금 해제 상태:** 모든 잠금 오버레이(🔒) 및 페이월 유도 요소를 완전 제거하여 완벽한 시각적 해방감과 유료 플랜만의 심미성을 극대화한다.
