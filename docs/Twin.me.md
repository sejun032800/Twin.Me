📑 Twin.me 기능 명세서 (SRS)
Product Version: 2.0 (Mirror & Archive Pivot)

Target Platform: React Native (iOS / Android / Web Preview)

1. 서비스 개요 및 목적 (Product Perspective & Intent)
기본 철학: Twin.me는 연인 간의 갈등에 개입하여 답변을 대신 작성해 주거나 상대방의 심리를 함부로 재단하는 '대리인(Agent)'이 아닙니다. 오직 유저 개인이 자신의 대화 습관을 객관적으로 돌아보게 만드는 '거울(Mirror)'이자, 휘발되는 카카오톡 대화 속에서 연인이 나에게 건넨 예쁜 말들을 온전히 보존하는 '감성 아카이브(Archive)'입니다.

Value Proposition (가치 제안): 1.  상대방 모르게 '나 혼자 보는' 은밀한 연애 오답 노트 및 대화 매너 교정.
2.  카톡 텍스트 더미 속에서 진주 같은 다정한 문장들을 추출해 박제하는 폴라로이드 월.



1. 온보딩 및 데이터 가공 모듈 (Onboarding & Data Ingestion)
기능 ID: FUN-ONB-001 (기초 정보 입력 및 커플 매칭)
기능 정의: 유저 가입 및 초대코드를 통한 1:1 커플 격리 서버 생성.
화면 및 UI 연출: 딥 미드나잇 인디고(#0A0D1A) 배경, 토스 재질의 미니멀 입력 폼. 초대코드 입력 성공 시 두 나선(DNA)이 자석처럼 결합하는 햅틱 진동 연출.
상세 로직:
유저 인증 완료 후 이름, 성별, MBTI 및 애니어그램 유형(모를 경우 '모름' 체크) 수집.
먼저 가입한 유저(A)에게 Unique ID 기반 8자리 초대코드 생성 및 카카오톡 공유 기능 제공.
상대 유저(B)가 코드 입력 시, 시스템은 DB 내 두 유저를 Couple_ID로 묶고 독립된 1:1 격리 네트워크(Shared Memory)를 할당함.
기능 ID: FUN-ONB-002 (온디바이스 카톡 말투 학습 및 필터링)
기능 정의: 카카오톡 .txt 대화 파일을 안전하게 전처리하여 내 말투 유전자 추출.
화면 및 UI 연출: 게이지 바가 차오르며 "개인정보 보호를 위해 기기 내부에서 상대방 대화 파기 중..." 문구 노출. 학습 완료 시 시그니처 드립(TOP 3) 카드 팝업.
상세 로직:
iOS(아이폰): 배포된 '공식 단축어매크로'를 통해 지정된 시간에 카톡 대화방 자동 백업 및 파일 전송 API 호출.
Android & 수동: 유저가 직접 내보낸 .txt 파일을 인앱 업로드.
정형 파싱 (Regex 로컬 엔진): 클라이언트 단(스마트폰 내부)에서 [상대방 이름]으로 시작하는 행 전체를 무조건 삭제(Drop)하여 서버 전송 전 원천 차단. 전화번호, 계좌번호 등 민감정보 마스킹(***).
말투 추출: 오직 본인의 대화 텍스트 중 자주 쓰인 감탄사, 어미, 비속어 빈도를 분석하여 LLM 프롬프트 주입용 페르소나 데이터 셋 빌드. 원본 파일은 기기 및 서버에서 즉시 영구 파기.
2. 메인 탭: 연애 대시보드 (Main Home)
기능 ID: FUN-HOM-001 (상단 정확도 배너 및 10분 인터뷰 퀘스트)
기능 정의: 가입 직후 AI의 초기 상태를 고지하고 심층 인터뷰 참여를 자발적으로 유도.
화면 및 UI 연출: 은은한 바이올렛 톤의 50% 진행 바 배너. 10분 대화 완료 시 배너가 연기처럼 사라지는 달성감 연출.
상세 로직:
카톡 데이터만 업로드된 초기 상태에서는 배너에 "현재 AI 정확도: 50%" 문구와 [🎙️ 10분 인터뷰로 정확도 95% 올리기] 버튼 노출.
버튼 터치 시 전체 화면이 [Twin.me 성향 분석 팀] 전화 수신 UI로 전환되며 음성 인터뷰(Realtime API/STT) 세션 개시.
사전 입력받은 MBTI/애니어그램 데이터를 Context로 활용하여 AI 면접관이 맞춤형 연애 가치관 질문(방어기제, 서운함 지수 등) 투척.
인터뷰 완료 시 성향 매트릭스 점수가 합산되어 정확도가 95%로 해제되며, 홈 배너는 영구 소멸. 해당 지표는 설정 탭의 [마이 트윈 AI 관리 센터]로 이식됨.
기능 ID: FUN-HOM-002 (실시간 상태 바 & 추억 링)
기능 정의: 상대방의 실시간 컨텍스트 파악 및 과거 데이트 아카이빙.
화면 및 UI 연출: X(트위터) 트렌드 UI 스타일의 상단 텍스트 바 + 인스타 스토리 하이라이트 형태의 동그란 링 아카이브.
상세 로직:
연인 AI가 백그라운드에서 분석한 상대방의 현재 심리/스케줄 상태를 X 트렌드 해시태그 형태로 실시간 동기화노출 (ex. #오늘부장님잔소리폭발).
하단 추억 링 터치 시 팝업 형태로 해당 데이트 당시의 베스트 컷과 서로의 OOTD가 피드로 가볍게 노출됨.
👨‍💼 PM의 데이터 베이스(DB) 설계 제언
위 명세를 구현하기 위해, 우리 시스템은 유저 한 명당 다음 3가지 데이터 칩을 독립적으로 관리해야 합니다.

User_Tone_Vector (카톡에서 추출한 문체, 시그니처 드립 데이터)
User_Persona_Matrix (10분 인터뷰 + MBTI/애니어그램으로 정제된 심리/방어기제 점수)
Shared_Memory_DB (커플 ID로 격리된 둘만의 벡터 DB - 대화 맥락, 위시리스트, 미쉐린 정보 적재)

3. 채팅 탭 모듈 (Twin Chat Module)

기능 ID: FUN-CHA-001 (인스타 DM 스타일 비주얼 분리 및 타이핑 호흡)
[System 2] 홈 탭: 마이 미러 대시보드 (My Mirror Dashboard)
기능 의도: 연인에게 노출되지 않는 오직 나만을 위한 소통 거울 룸. 내가 연인에게 보낸 카톡의 어조와 습관을 AI 관점에서 피드백하여 자가 교정을 돕습니다.

세부 명세:
나의 대화 톤 피드백 리포트: 최근 대화 중 은연중에 사용한 압박성 표현, 단답형 어조, 혹은 감정적 단어의 빈도수를 인포그래픽(게이지 트랙)으로 시각화.

소통 매너리즘 방지 가이드: "최근 3일간 서영이의 질문에 공감 수치보다 해결책 제시 비중이 80% 높았습니다. 오늘은 조용히 든든한 공감 한 줄을 건네보는 건 어떨까요?" 같은 나의 행동 지침 지적 큐레이션 노출.

주의 사항: 상대방이 어떻게 해야 하는지가 아닌, '내가 어떻게 해야 하는가'에 초점을 맞춤.

기능 정의: 연인 및 AI 에이전트들과의 채팅 목록 격리 및 실제 인간과 유사한 분할 타이핑 전송.
화면 및 UI 연출: 최상단 연인 방 고정. 그 밑으로 트윈 AI 블루(#38BDF8) 고유 뱃지가 부착된 AI방 배치. 말풍선 전송 시 글자 수와 타수에 비례한 시간차(Delay) 애니메이션 및 '말하는 중...' UI 표출.

상세 로직:
인풋 가공 (버퍼링 타임아웃): 사용자가 카톡처럼 짧게 연타로 메시지를 보낼 경우(엔터), 시스템은 즉시 답변하지 않고 **2.5초의 버퍼(대기 시간)**를 유지함. 2.5초 이내에 추가 인풋이 들어오면 하나의 문장 맥락으로 합산(Aggregation)한 뒤 LLM 엔드포인트를 호출함.
아웃풋 가공 (Output Splitter): AI가 생성한 통문장 답변을 문장 부호(. , ? !) 또는 글자 수 기준으로 쪼갬. 진짜 남친/여친이 타자를 치는 호흡에 맞춰 0.5초~1.5초의 시간차를 두고 말풍선을 분할 전송함.

기능 ID: FUN-CHA-002 (실시간 말투 교정 리포트 및 피드백 룹)
기능 정의: 대화 중 이질감이 느껴지는 AI의 답변을 유저가 직접 실시간으로 교정 및 길들이기.
화면 및 UI 연출: AI 말풍선 롱 프레스(Long Press) 시 하단 시트(Bottom Sheet) 슬라이드 업.
상세 로직:
유저가 AI의 말풍선을 롱 프레스하여 **[🤖 말투 교정하기]**를 터치하면, 세부 피드백 옵션(너무 다정함 / 너무 딱딱함 / 유머 코드 안 맞음) 팝업.
유저가 옵션을 선택해 제출하면, 시스템은 가중치(Weight)를 조정하여 즉시 문맥을 수정한 답변을 재생성하고 프롬프트 메모리에 즉각 반영함.
이 말투 교정 로그가 누적 3회 이상 발생 시, 분석가 '트윈이' 오피셜 챗봇이 "일일이 고치기 귀찮으시다면, 메인 탭에서 딱 10분만 전화를 받아보세요!"라는 개인화 넛지 알림을 트리거함.
기능 ID: FUN-CHA-003 (긴급 성찰 경보: Mirroring Crisis Mode)
기능 정의:
실제 커플 간의 텍스트 대화에서 파국/단절의 징후가 감지될 때, 백그라운드 AI가 유저의 최근 대화 데이터를 정밀 분석합니다. 이를 통해 타협안을 내려주는 것이 아니라, 유저가 흥분 상태에서 놓친 '나의 잘못된 대화 패턴'과 '상대방이 느꼈을 상처의 깊이'를 거울처럼 비춰주어 유저 스스로의 성찰과 반성을 유도하는 긴급 브레이크 시스템입니다.

화면 및 UI 연출:

기존의 공격적인 사이렌 레드 대신, 깊고 차분한 딥 인디고/차콜 베이스에 경고성 네온 와인 컬러가 감도는 전면 글래스모피즘 팝업 연출.

화면 타이틀: "잠시 대화를 멈추고 거울을 바라보세요"

국가 재난문자 특유의 무거운 진동을 커스텀하여, 유저가 이성을 찾을 수 있도록 묵직하고 은은한 투-둠, 투-둠 햅틱 진동 패턴을 약하게 트리거.

상세 로직:

위기 단계 진입 조건: 실제 커플 채팅 데이터에서 갈등 임계 키워드("헤어져", "짜증나", "그만하자")가 발각되거나, 3시간 이상의 날카로운 의도적 대화 단절(Deadlock) 발생 시 백그라운드 엔진 가동.

AI 성찰 매트릭스 연산: AI 객체가 최근 24시간 동안 유저가 보낸 문장들을 추출하여 소통 심리학 모델을 기반으로 난폭성, 방어 기제, 공감 결여 수치를 연산.

'반성의 거울' 리포트 도출 (핵심 피벗):

연인에게 보낼 타협안 텍스트를 짜주는 로직을 완전 폐기.

오직 유저 혼자만 보는 팝업 내부에 "방금 전 서영이의 대화에 당신이 보낸 문장은 서영이에게 단순한 반박을 넘어 깊은 거절감을 주었을 확률이 84%입니다. 당신은 대화를 빨리 끝내기 위해 서영이의 서운함을 '징징거림'으로 치부하지 않았나요?" 같은 뼈아픈 자기 성찰 문구 도출.

행동 지침 미션 및 액션:

연인의 폰에는 아무것도 가지 않음 (은밀한 자가 교정).

유저의 폰에만 스스로의 감정을 가라앉히는 [진정하고 나-전달법(I-Message)으로 진심 전하기] 가이드와 함께, [👉 내 대화 습관 인정하고 돌아가기] 단일 액션 버튼 배치.

기능 ID: FUN-CHA-004 (AI 데이트 뮤즈: 무드 맞춤형 코스 큐레이션)
기능 정의:
사용자가 오늘 입은 OOTD, 현재 무드, 느끼고 싶은 감성 등을 선택하면, 그동안 앱 내에 축적된 우리 커플의 데이트 데이터(DateCourse 히스토리)를 기반으로 외부 무료 LLM API를 호출하여 우리 커플의 취향을 반영한 맞춤형 데이트 코스를 자율 큐레이션하는 시스템입니다.

화면 및 UI 연출:

🗺️ 데이트 지도 페이지 우측 하단(기존 + FAB 버튼 위쪽)에 영롱한 오로라 그라데이션 피막이 감도는 [AI 데이트 뮤즈] 플로팅 버튼 배치.

버튼 터치 시, 반투명 글래스모피즘 바텀 시트가 슬라이드 업되며 [오늘의 OOTD 스타일(ex. 캐주얼, 시크, 페미닌), 현재 무드(ex. 차분함, 신남, 로맨틱)]을 선택할 수 있는 미니멀 칩셋 폼 노출.

코스 생성 중에는 지도 중앙에 은은한 별가루 파티클 애니메이션 표출.

생성 완료 시, 제안된 코스들이 카카오 지도 위에 영롱한 핑크빛 추천 경로 점선(kakao.maps.Polyline)으로 연결되며 맵 위에 프리뷰 렌더링.

상세 로직:

컨텍스트 어그리게이션 (Context Aggregation): 유저가 바텀 시트에서 선택한 현재 입력값과 함께, AppContext 내에 저장되어 있는 [우리 커플이 그동안 방문했던 장소 목록, 각자 매긴 별점 점수, 자주 가던 선호 지역] 데이터를 하나의 프롬프트 데이터셋으로 정밀 패킹함.

무료 LLM 오케스트레이터 연동 (Free AI Wrapper): 외부 무료 티어 API(예: 가상 허브 엔드포인트 또는 무료 오픈소스 모델 래퍼)를 비동기 호출. 패킹된 기존 데이터 패턴을 학습 데이터(Few-Shot Context)로 주입하여, "기존에 별점 5점을 주었던 조용한 카페 감성을 선호하되, 오늘 선택한 '시크 OOTD'에 어울리는 새로운 연계 코스 3곳"을 추천 사유와 함께 JSON 배열로 받아옴.

데이터 벌크 인서트 및 아카이브: 제안된 코스 카드 하단에 [👉 이 코스로 데이트 확정하기] 버튼 배치. 유저가 이를 클릭하면 제안된 장소들이 DateCourse 전역 상태 배열에 즉각 일괄 삽입(Bulk Insert)되어, 추후 실제 방문 후 별점과 후기를 남길 수 있는 대기 상태로 지도 위에 핀이 자동 생성됨.

동적 뷰포트 스크롤: 나선 모드 스크롤 시 카메라는 3D 회전 각도를 유지하며 램프를 타고 올라가는 입체적 시점 제어 제공.
